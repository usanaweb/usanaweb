Navegação para Dispositivos Móveis

Paula Oliveira

12 de Dezembro de 2024

Tempo de leitura:

7 min

Resumo: Atualmente, as pessoas utilizam mais o celular para acessar a internet em vez do computador, aliás algumas não possuem computador, seja pelo custo ou por não sentirem necessidade. Por tanto, é necessário que o seu site entregue uma boa usabilidade e experiência para o usuário, pois provavelmente é por este meio que ele irá conhecer sua empresa.

  • A Experiência do Usuário em Sites para Dispositivos Móveis

    Em 2007, quando Steve Jobs lançou o primeiro iPhone, o mercado de dispositivos móveis começou a ganhar mais força, enquanto a navegação web para celular ainda era primitiva. Através dos anos que se passaram, a experiência do usuário de dispositivo móvel foi se consolidando e padrões foram estabelecidos. No Brasil, atualmente, cerca de 92,5% dos lares brasileiros possuem acesso à internet, sendo que 96,7% possuem celular. Sendo assim, é essencial que o seu site tenha uma boa usabilidade e experiência do usuário.

  • Como melhorar a Usabilidade do Site Móvel

    Como citado no tópico anterior, padrões para o design móvel de sites estão estabelecidos, dentre eles estão:

    a) Barra de navegação: assim como na versão para computador, ela se localiza na parte superior da tela. Deve ser usada quando o número de opções seja no máximo cinco. Caso o número de opções seja maior que cinco, uma rolagem horizontal pode ser utilizada, desde que as opções relacionem entre si;


    b) Barra de Guias: usada também para navegar entre as opções, mas diferente da barra de navegação que some depois da rolagem, a barra de guias é fixa na tela. Em relação ao número de opções e à rolagem horizontal, seguem as mesmas indicações citadas na barra de navegação;


    c) Menu hambúrguer: apesar de usuários mais velhos não estarem familiarizados com o ícone de menu hambúrguer, para pessoas da Geração Y e à frente ele faz parte do seu dia a dia. No entanto, para o site ser acessível para todos, complemente a navegação adicionando outros artifícios, como breadcrumbs, links na página inicial e rodapés;


    d) Rolagem infinita: ela surgiu principalmente por conta das redes sociais, mas o seu uso gera mais problemas de usabilidade do que benefícios para o site. Por tanto, se no seu site o usuário geralmente procura algo específico, como um artigo, utiliza para comparar itens ou verifica apenas itens localizados no topo da página, como resultados de pesquisas, evite a rolagem infinita. Se o seu site não se enquadra nestes casos, utilize a rolagem infinita interrompida pelo botão “carregar mais”, através desse recurso você evita problemas comuns, como lentidão no carregamento da página, não conseguir acessar o rodapé ou navegar pelo teclado;


    e) Carrossel: permite que vários conteúdos sejam mostrados sem precisar rolar a página, resolvendo o problema de espaço da tela do celular. No entanto, alguns cuidados devem ser tomados para não ocorrer erros de usabilidade;


    • Não parece um carrossel: utilize recursos como os pontos abaixo da imagem, os itens anteriores e posteriores devem aparecer pela metade e textos cortados. Esses recursos transmitem para o usuário a sensação de continuidade;


    • Gestos de toque: dependendo da marca do celular, alguns gestos realizados na tela podem ativar funções específicas no celular, atrapalhando a usabilidade do carrossel. No caso do iPhone X, quando o usuário desliza perto da borda da tela, ativa a função de alternar entre os aplicativos. Para contornar essa situação, utilize setas para clicar e passar os slides, ou insira um espaço maior nas bordas laterais do site para que a função não seja ativada;


    f) Alvos de toque: deve se atentar ao tamanho do elemento clicável e ao espaçamento adequado dos outros elementos para não tocar o item errado. O recomendado é que o alvo de toque ocupe 1 cm x 1 cm na tela, mas deve considerar se o usuário vai utilizar o site enquanto se movimenta, se o usuário é um idoso ou criança que não possui uma boa precisão no toque. Nestes casos, utilize um elemento de 2 cm x 2 cm e realize testes com os usuários para verificar se um tamanho maior é necessário;


    f) Acordeões: um elemento que se expande para mostrar mais informações para o usuário, eles podem ser usados para filtros, categorias, formulários, etc. Para que eles funcionem corretamente sem impactar na usabilidade, deve-se considerar dois pontos:


    • Não mover o acordeão para o topo da página: é comum que ao usuário abrir o acordeão, ele suba para o topo da página. Essa ação pode confundir o usuário, que pode achar que uma nova página foi aberta e tentar voltar para a anterior. Para corrigir esse erro, faça com que, quando o acordeão estiver expandido e o usuário clicar em voltar, o acordeão seja fechado;


    • Conteúdo extenso: quando o texto contido no acordeão é muito extenso, utilize a opção de deixar o acordeão fixo na tela ou insira um elemento flutuante para fechar. Além disso, você pode usar o botão voltar como uma forma de fechar o acordeão;


  • Conclusão

    Em relação à navegação tradicional pelo computador, a navegação web móvel é mais recente e precisa de mais pesquisas para compreender todos os requisitos necessários para construir um site com a melhor usabilidade. No entanto, atualmente já possuímos padrões estabelecidos e consolidados, como o menu hambúrguer, a barra de guias e a rolagem infinita. Enquanto, elementos comuns na versão para computador foram adaptados para o celular, como o carrossel, acordeões e áreas clicáveis. Devido o celular ser o meio mais utilizado para acessar sites, todos esses pontos devem ser considerados para entregar uma experiência móvel adequada para o usuário.
