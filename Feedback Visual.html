Resumo: Quando o usuário não consegue ter o controle da situação ao navegar pelo site, ele deixa de utilizá-lo. Para que isso não ocorra, utilize elementos visuais na interface para comunicar o usuário em tempo real o que está acontecendo. Neste artigo mostraremos alguns exemplos que você pode utilizar em seu site.

  • Como construir um Bom Feedback Visual

    O feedback visual é uma forma da interface esclarecer para o usuário o que está acontecendo em tempo real. Essa comunicação da interface com o usuário é importante para que ele se sinta confiante ao utilizar o site. Manter o estado do sistema sempre visível é uma das heurísticas de usabilidade. Para construir um site com um bom feedback visual, é importante considerar os seguintes fatores:

    a) Contexto: Onde Estou? Quando o usuário acessa uma página, ele precisa se localizar no site, assim ele consegue compreender onde ele está e para onde ele deseja ir. Para fornecer informações de localização para o usuário, você pode utilizar o Breadcrumbs (trilha de migalhas de pães), ele permite que o usuário volte e avance pelo caminho que ele já fez. Além disso, pode usar títulos nas páginas para informar sobre o conteúdo da página. Outro recurso é o menu, quando a categoria é selecionada, ela é destacada, seja por um texto com negrito, cor de fundo ou uma borda em volta do texto;


    b) Status atual: O ser humano gosta de ter o controle da situação, por isso, quando a interface não mostra um feedback logo após ele realizar uma ação, ele se sente inseguro e ansioso. Afinal, o que está acontecendo? No site, é utilizado o carregamento da página para o usuário ter certeza de que o clique no elemento clicável foi reconhecido;


    c) Status futuro: O que vai acontecer depois? Depois que o usuário executa uma ação na página, ele pode querer entender o que vai acontecer depois. Nestes casos, o uso de texto e notificações é o mais comum. Por exemplo, a pessoa comprou um produto no site, após realizar a compra, ela quer saber como vai acompanhar seu pedido: será no site? Através do site da transportadora?;


    d) Resultado: O que acabou de acontecer? Quando o usuário realiza uma ação, ele sente a necessidade de saber se ocorreu com sucesso. Sendo assim, a interface deve, por meio de notificações e textos, informar o usuário que sua ação foi realizada com sucesso, e não somente informar quando ocorrem erros. No site, por exemplo, quando o usuário envia os dados do formulário, ele quer ter o retorno de que o formulário foi realmente enviado, porque se não houver essa informação, o usuário pode entender que houve algum erro;


  • Qual a Melhor Forma de Fornecer o Feedback

    Após compreender os seguintes fatores que devem ser considerados para o seu site ter um bom feedback visual, vamos falar sobre os principais meios que essa comunicação acontece, que você pode utilizar:

    a) Indicadores: são uma forma de informar o usuário alguma informação importante, através do ícone, do texto em negrito, por um tamanho, por uma animação. Os indicadores são utilizados conforme o contexto, se alteram conforme a condição e não exigem uma ação por parte do usuário. Para saber quando inserir um indicador em sua página, pense: O quão relevante ele é para o usuário? Um exemplo de indicador é o site informar quanto tempo o usuário leva para ler o artigo da página, através de um texto em negrito com um ícone de relógio ao lado;


    b) Validações: são as mensagens de erro, que podem ocorrer quando um campo de formulário não foi preenchido ou que houve algum erro ao enviar um arquivo, etc. As validações são contextuais e o usuário precisa realizar uma ação para retirar a mensagem, e assim concluir que o erro foi corrigido. Para este tipo de feedback, pode ser utilizada uma mensagem próxima ao elemento que precisa ser corrigido, utilizando o vermelho, uma borda ao redor do elemento e/ou um ícone. Além disso, a mensagem deve ser clara sobre como corrigir o erro, em vez de utilizar “Campo em Branco” use “Preencha o campo de telefone para contato”;


    c) Notificações: são um meio de informar o usuário sobre acontecimentos gerais do sistema, não estão ligados necessariamente a uma ação do usuário ou contexto. As notificações podem solicitar uma ação necessária, sendo mais intrusiva, por meio de um pop-up que interrompe a ação do usuário para que aquela situação seja resolvida, são informações para alertar sobre algo que impacte o usuário negativamente. Enquanto, na notificação passiva, ela é utilizada para fornecer uma informação que pode ser relevante para o usuário, mas não exige uma ação por parte dele, por isso ela é menos invasiva, aparecendo no canto da tela por um ícone com texto ou um popover que não interrompe o usuário. Um exemplo destes dois casos é um pop-up para o usuário preencher dados importantes que faltaram para o cadastro (notificação emergente), e uma notificação sobre um cupom de desconto para itens eletrônicos (notificação passiva);


  • Conclusão

    O feedback visual é um recurso que ajuda o usuário a se sentir mais seguro e confiante ao navegar pelo site. O feedback pode ser comunicado para o usuário de diversas maneiras, como: breadcrumbs, texto em negrito, alteração da cor de fundo do elemento, um pop-up intrusivo ou passivo, mensagens de erro e ícones. Além disso, devem ser aplicados geralmente em um contexto, para não sobrecarregar o usuário com informações que são irrelevantes naquele momento. Os feedbacks podem ser aplicados para informar em tempo real o que está acontecendo, o que vai acontecer depois e se a ação foi realizada com sucesso. Por fim, o feedback visual ajuda a indicar para o usuário informações relevantes, a validar as informações para serem enviadas corretamente, e notificar sobre ações urgentes que o usuário deve fazer naquele momento.
