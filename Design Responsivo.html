  • Porque ter um Site com Design Responsivo?

    O design responsivo é uma abordagem do desenvolvimento web para que, através do mesmo código HTML, o site seja utilizado em diferentes tamanhos de tela. A adaptação ocorre por meio dos pontos de interrupção, cada ponto de interrupção é baseado no tamanho da largura da tela. Na ferramenta Bootstrap, de que falaremos no próximo tópico, ela divide os pontos de interrupção em sm (pequeno), md (médio), lg (largo), xl (grande) e xxl (extra-grande). Por exemplo, um celular ficaria no ponto de interrupção sm (pequeno), enquanto o computador seria do ponto lg e acima. Ou seja, ter um site responsivo é importante para:

    a) Mobile Fist: segundo dados do IBGE de 2023, cerca de 96,7% dos lares brasileiros possuíam celular e 92,5% tinham acesso à internet. Por tanto, no momento do desenvolvimento do site, primeiramente deve ser pensado no layout do celular, por ser através dele que os usuários vão ter acesso, e o site responsivo permite que o seu site funcione corretamente em telas menores;


    b) Experiência: Implica diretamente na experiência do usuário, porque ele pode acessar o conteúdo e mudar para outro sem dificuldades. Por exemplo, a pessoa pode fazer uma pesquisa no celular enquanto está no ônibus e, quando chegar em casa, ela pode continuar acessando no computador para escrever os dados em um artigo;


    c) Gestão: sem o site responsivo, para que você conseguisse acessar o seu público que utiliza dispositivos com telas de tamanhos diferentes, seria necessário ter um site para cada tipo de tela: celular, tablet, laptop, etc. Enquanto no design responsivo você consegue o acesso em telas diferentes por meio de um único código e alterar elementos que impactam em todas as telas por meio deste único código HTML;


  • Criando um Site Responsivo

    Agora que compreendemos a importância do design responsivo, vamos explicar os principais pontos que devem ser considerados para a criação do layout:

    a) Conteúdo: deve ser priorizado as informações importantes que pela tela maior do computador podem ser acessadas facilmente, mas difíceis de acessar em uma tela menor de celular, sendo necessário rolar a página mais vezes para ter acesso. Neste caso, faça a troca de posição dos elementos para que o usuário do celular tenha o mesmo acesso fácil que o usuário do computador;


    b) Desempenho: por ser o mesmo código, a mesma informação enviada para o computador é a mesma para o celular, o que pode prejudicar a experiência do usuário de celular caso ele esteja em um local com internet instável. Existem alguns servidores que trabalham com sites adaptativos, enviando conforme identificado o dispositivo um HTML personalizado para a tela, e assim não deixar o carregamento lento da página em celulares. No entanto, você pode fazer o teste de utilizar o seu site em lugares que você sabe que possui uma conexão ruim, para verificar como o site se comporta e refinar para os elementos ficarem mais leves. Para saber mais sobre o desempenho do seu site neste artigo descrevemos mais sobre uma ferramenta que verifica o desempenho e instrui você sobre os principais problemas que estão gerando lentidão;


    c) Imagens: por ser um elemento de tamanho fixo, elas podem gerar problemas de layout. Quando são infográficos e motion design, existe a possibilidade de cortes para uma diagramação dinâmica, mas no caso de e-commerces, elas precisam destacar e apresentar o produto. Para resolver esse problema, é possível utilizar os recursos do design responsivo para a imagem ocupar 100% das grades do grid, para que ela se adapte em diversas telas;


    d) Animação e Vídeo: assim como a imagem, o vídeo e animações são mais limitados. No vídeo, é possível deixar o elemento com responsividade, assim como citamos anteriormente para a imagem. No caso das animações de elementos, é preciso realizar um teste para verificar como está o desempenho das animações em telas pequenas e grandes. Em alguns casos, é necessário modificar a animação, por exemplo, se a animação ocorre na horizontal em telas maiores, em telas pequenas talvez seja necessário que a animação ocorra na vertical;


    e) Tipografia: é um dos elementos mais fáceis de tornar responsivo com o site, pois apenas varia o tamanho do texto e peso do tipo, focando em deixar o texto legível em todos os tamanhos de tela. O tamanho mínimo para os textos deve ser de 10pts;


    f) Chamar para Ação: conhecido por CTA, são partes do site que são interativas e chamam a atenção do usuário para realizar uma ação. O elemento CTA deve estar disponível e visível em todas as telas, sendo realizados testes para observar se o elemento está cumprindo sua função;

    Para tornar o site responsivo, você pode fazer isso manualmente, ou utilizar ferramentas que podem facilitar o processo. A ferramenta mais utilizada é o Bootstrap, a plataforma oferece uma integração via CDN para utilizar os atributos prontos para facilitar o processo, sem perder a possibilidade de personalizar a sua maneira. Não sabe como funciona o Bootstrap? O profissional Ricardo Maroquio disponibilizou no seu canal do YouTube um curso gratuito sobre o Bootstrap 5, acesse o curso!

  • Conclusão

    O Design Responsivo é uma maneira de tornar o layout do seu site adaptado para todas as telas, permitindo alcançar todo o público que pode alternar entre o uso de dispositivos sem perder a informação. Sendo assim, é importante verificar se o site está funcionando em todas as telas adequadamente, desde o conteúdo, como imagem, vídeo e tipografia. Além disso, os elementos que chamam a atenção do usuário para uma ação devem estar em todas as telas em locais que vão ser vistos, porque um elemento que está visível no computador talvez não esteja na tela do celular. Por fim, o desempenho do site também deve ser responsivo. Uma imagem sendo carregada para o computador pode ser pesada para carregar em um celular. Para verificar o desempenho de carregamento do site, você pode realizar testes manuais ou utilizar ferramentas de validação .
