Botões

Resumo: Os botões auxiliam o usuário na navegação, em tomadas de decisões, a criar ações (criar um e-mail para enviar ao chefe), eles estão presentes em vários momentos da interface. Por tanto, explicaremos neste artigo como um botão se estrutura e quais as principais características que você deve se atentar para não prejudicar a usabilidade da interface.

  • Anatomia do botão

    O botão está presente por toda a interface do site, sendo utilizado pelo usuário para navegar e enviar informações para o servidor. Por isso, você deve se atentar se o usuário consegue identificar facilmente na interface e se ele consegue realmente utilizá-lo. Para que isso aconteça, é importante que você conheça os elementos que compõem o botão para poder aplicar as dicas do artigo:

    a) Cor: utiliza cores da identidade visual da empresa, usado para chamar atenção para a ação do botão;


    b) Forma: os botões possuem geralmente formato retangular com bordas arredondadas;


    c) Tipografia: também é utilizada a tipografia da identidade visual da marca para manter a consistência;


    d) Margem: espaço mínimo que deve existir entre o botão e os elementos do site;


    d) Ícones: alguns botões podem conter ícones para indicar ações que os usuários estão acostumados a associar ao ícone, por exemplo, o disquete associado à ação de salvar informações;


  • Tipos de botões

    Compreendendo as características principais dos botões, aprofundaremos a estilização dessas características nos vários tipos de botões que existem:

    a) Chamada para Ação: é o botão utilizado para instruir ao usuário o que ele deve fazer primeiro na tela, por exemplo, nas páginas iniciais é comum logo no início do conteúdo ter um botão para conhecer mais sobre os serviços, a empresa, etc.;


    b) Primário: utilizado com maior frequência na interface por ser usado para executar ações, como confirmar ações ou criar uma ação. Por exemplo, o botão de “criar um novo e-mail” é um botão primário. Esse tipo de botão deve ser fácil de ser identificado, a cor dele deve chamar mais a atenção que o restante da interface;


    c) Secundário: usado geralmente junto de um botão primário para o usuário decidir entre duas ações. Por ser um botão secundário, não pode utilizar uma cor que compete com o botão primário, use cores mais claras, menos saturadas, menos o cinza para que o usuário não associe que o botão está desabilitado. Um exemplo, é na imagem anterior em que o botão secundária é apenas um texto em negrito em vermelho que chama menos atenção que o botão primário com fundo vermelho;


    d) Botões fantasmas: são botões sem preenchimento, apenas com borda e tipografia. Como seu próprio nome diz, ele não é um botão que chama a atenção do usuário, mas é bastante utilizado para deixar o visual da interface minimalista. Utilize esse tipo de botão como secundário;


    e) Botões com ícones: o padrão utilizado normalmente é que o ícone, quando está ao lado esquerdo, está associado à ação do botão e quando está do lado direito, é para representar continuidade. Por exemplo, um botão escrito “enviar” e um ícone de flecha ao lado direito indica que a mensagem será enviada. Quando utilizar o ícone no botão, procure deixar ele centralizado horizontalmente com o texto, além de um espaço entre texto e ícone;


    f) Botão Voltar ao topo: diferente do convencional, o botão de voltar ao topo pode ser redondo ou quadrado, sendo utilizado, como o próprio nome indica, para voltar ao topo da página. Esse botão é indicado para conteúdos que passam de 4 telas. Além disso, deve estar ao lado inferior direito da tela e ele não pode cobrir outros elementos;


  • Pontos para considerar de usabilidade

    Além de alguns pontos já citados anteriormente em relação aos cuidados que se deve ter ao utilizar alguns tipos de botões, algumas características devem ser consideradas enquanto para que o apelo visual não prejudique a usabilidade do botão:

    a) Contraste: para o botão ser utilizado, ele precisa ser primeiramente visto e compreendido. Quando o texto e o fundo não possuem um contraste de cor suficiente para ser legível, o botão perde sua utilidade. Para saber se o botão do seu site tem contraste adequado, acesse este artigo em que indicamos uma ferramenta específica para essa validação. Além disso, o botão deve comunicar a ação que o usuário realizou, como mudar de cor quando o usuário clica, quando passa o mouse por cima e quando algo está sendo carregado, etc.;


    b) Tom de voz: saber comunicar com o usuário é fundamental para ele compreender o conteúdo da página. No caso dos botões, é essencial que ele comunique para o usuário exatamente o que ele faz ao clicar. Por exemplo, em vez de utilizar “Acesse aqui” use “Baixe o relatório”, ou seja, especifique a ação do botão. Não sabe qual o tom de voz do seu site? Acesse este artigo para aprender como se comunicar com seu usuário;


    c) Estilos: existem vários estilos de botões, mas listaremos dois estilos que são os mais utilizados e que não geram tantos problemas de usabilidade como:


    • Almost Flat: são como os botões flat, mas com uma sombra em volta para gerar tridimensionalidade, para o usuário conseguir identificar como um elemento clicável;


    • Gradiente: uma alternativa ao flat, mas que deve ser utilizado com o devido cuidado para gerar contraste entre fundo e texto. Assim como flat, prefira inserir uma leve sombra para trazer a tridimensionalidade, para ser visualmente identificado como botão;


    Assim como citados anteriormente, existem vários estilos como o próprio flat, gradiente iluminado e Skeumórfico, mas eles geram problemas de usabilidade ou estão ultrapassados, por isso não indicamos no artigo.


  • Conclusão

    Os botões são elementos que vão estar presentes em todas as páginas do seu site, seja para enviar um formulário, acessar uma página ou iniciar uma ação. Sendo assim, um elemento importante como os botões merecem um cuidado em relação ao tom de voz, que deve ser objetivo, sem gerar dúvidas no usuário sobre sua função. O contraste entre texto e fundo deve estar correto, e as cores e estilos devem ser condizentes com sua função, seja primária ou secundária, além de não prejudicarem a usabilidade.
